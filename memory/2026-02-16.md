## Critical Lesson: NEVER Commit to Main

**What I did wrong:**
- Committed mind folder directly to main
- Committed skill updates directly to main
- Violated the "never commit to main" rule that was in AGENTS.md

**User correction:**
> "whenever you do anything you gotta do it in a branch and when codesync you do a pr"

**Correct workflow (now documented in skills):**
1. **Create feature branch FIRST:** `git checkout -b feat/description`
2. **Do the work**
3. **Run CodeSync checks** (type check, unit tests, e2e tests)
4. **Commit to branch**
5. **Push branch:** `git push origin feat/description`
6. **Create PR:** `gh pr create`
7. **Merge via PR** (not direct commit)

**NEVER:**
- ❌ `git commit` on main
- ❌ `git push origin main`
- ❌ Direct commits to main branch

**Skills updated:**
- `mind/skills/codesync/CodeSync.md` — Added branch workflow, Step 0 (create branch), verification steps
- `mind/skills/mindsync/Mindsync.md` — Clarified CodeSync vs MindSync separation

## CodeSync vs MindSync Clarification

**CodeSync:**
- Commits **code** changes
- Feature branch → PR → Merge
- Tests must pass
- Git workflow

**MindSync:**
- Updates **mind** content (.md files)
- Direct edits (content, not code)
- No tests (documentation)
- Happens AFTER CodeSync completes

**Order:**
1. CodeSync → Commit code (branch → PR)
2. MindSync → Update mind files
