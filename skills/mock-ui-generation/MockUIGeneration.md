# Mock UI Generation

> Generate ASCII-based mock UI components inside markdown files for effective communication without building full HTML/CSS.

## Why Mock UI?

- **Fast iteration** - Sketch UI ideas in minutes, not hours
- **Universal readable** - Works in any text editor, terminal, or chat
- **Version controlled** - Diff-friendly like code
- **Accessible** - No design tools needed
- **Portable** - Copy-paste anywhere

## Sizing Guidelines

### Width Conventions

| Size | Characters | Usage |
|------|------------|-------|
| XS | 20 | Buttons, badges |
| SM | 30 | Input fields |
| MD | 40 | Cards, dialogs |
| LG | 60 | Modals, panels |
| XL | 80 | Full layouts |
| Full | 100+ | Wide tables |

### Height Conventions

| Lines | Usage |
|-------|-------|
| 3 | Button, badge |
| 5 | Input field |
| 7 | Small card |
| 10 | Medium card |
| 15 | Large panel |
| 20+ | Full page |

---

## Workflow

### Step 1: Choose Template
Select base template from library (card, modal, form, etc.)

### Step 2: Customize Content
Replace placeholder text with actual content

### Step 3: Add Components
Insert buttons, inputs, or other elements as needed

### Step 4: Add Status/Emoji
Include status indicators and action icons

### Step 5: Adjust Sizing
Resize to fit content naturally

### Step 6: Review
Check alignment and spacing visually

## Best Practices

1. **Keep it simple** - Don't over-detail
2. **Consistent sizing** - Align columns carefully
3. **Use emoji sparingly** - For status/key points only
4. **Test in monospace** - Ensure alignment
5. **Mobile-friendly** - Keep width under 80 chars
6. **Version control** - Easy to diff changes
7. **Document intent** - Add notes for complex interactions
